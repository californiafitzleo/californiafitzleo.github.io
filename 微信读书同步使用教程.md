# 微信读书划线同步 · 使用教程

## 方案概述

基于 Playwright 浏览器自动化，直接从微信读书网页版抓取划线数据，同步到 Hexo 博客的 Markdown 文件中。

- **数据流向**：微信读书网页 → Playwright 爬虫 → 博客 Markdown 文件
- **核心特点**：白名单机制、增量更新、手动内容保留、章节自动匹配

---

## 目录结构

```
scripts/weread-sync/
├── index.js      # 主入口（配置加载 + 同步流程）
├── scraper.js    # Playwright 爬虫（登录 + 抓取划线）
└── sync.js       # 章节匹配 + Markdown 处理

source/_data/
├── weread_mapping.yml   # 书籍映射配置（白名单）
├── weread-storage.json  # 登录状态（自动生成，勿手动编辑）
└── weread.styl          # 划线卡片样式
```

## 第一步：安装依赖

项目已在 `package.json` 中配置好 `playwright`，如果还没安装，执行：

```bash
npm install
```

安装 Playwright 浏览器（首次使用必须执行）：

```bash
npx playwright install chromium
```

---

## 第二步：扫码登录

在项目根目录执行：

```bash
npm run weread-login
```

会自动打开浏览器，跳转到微信读书网页版。**用微信扫码登录**，登录成功后浏览器会自动关闭，登录状态保存到 `source/_data/weread-storage.json`。

> 💡 登录状态有效期一般为几天到几周，过期后重新执行此命令即可。

---

## 第三步：配置书籍白名单

编辑 `source/_data/weread_mapping.yml`，配置需要同步的书籍。

### 配置字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| `weread_book_id` | 微信读书书籍 ID | `"39423423"` |
| `weread_book_title` | 书名（仅用于显示错误信息） | `"理想国"` |
| `blog_book_dir` | 博客中对应书籍的文件夹名 | `"Book-01"` |
| `sync` | 是否同步，`true` 同步 / `false` 暂停 | `true` |
| `chapters` | 手动章节映射（可选） | 见下方说明 |

### 获取微信读书书籍 ID

1. 打开 [微信读书网页版](https://weread.qq.com)
2. 打开某本书的详情页
3. 查看浏览器地址栏，URL 格式为：
   ```
   https://weread.qq.com/web/book/detail/{bookId}
   ```
4. `{bookId}` 就是书籍 ID

### 配置示例

```yaml
books:
  - weread_book_id: "39423423"
    weread_book_title: "理想国"
    blog_book_dir: "Book-01"
    sync: true
    chapters: []

  - weread_book_id: "12345678"
    weread_book_title: "百年孤独"
    blog_book_dir: "Book-03"
    sync: false
    chapters: []
```

## 第四步：章节匹配机制

### 自动匹配（推荐）

程序会自动根据文件名匹配章节，规则：
- 去掉文件名开头的数字编号（如 `01第一卷.md` → `第一卷`）
- 与微信读书的章节名精确比对

**只要文件名（去掉数字前缀后）和微信读书章节名完全一致，就能自动匹配。**

### 手动匹配

如果自动匹配失败（章节名不一致），可以在 `chapters` 中手动配置：

```yaml
books:
  - weread_book_id: "39423423"
    weread_book_title: "理想国"
    blog_book_dir: "Book-01"
    sync: true
    chapters:
      - weread_chapter: "第一章 财产与公道"
        blog_post_file: "01第一卷.md"
      - weread_chapter: "第二章 个人与国家"
        blog_post_file: "02第二卷.md"
```

## 第五步：执行同步

配置完成后，在项目根目录执行：

```bash
npm run weread-sync
```

| 参数 | 说明 |
|------|------|
| `--show` | 显示浏览器界面（调试用，默认后台运行） |
| `DEBUG=*` | 输出详细调试日志 |

示例：
```bash
# 显示浏览器界面运行
npm run weread-sync -- --show

# 调试模式
DEBUG=true npm run weread-sync
```

## 划线内容在 Markdown 中的结构

同步后，划线内容会添加到文章末尾，用特殊标记包裹：

```markdown
# 文章标题

正文内容...

<!-- WEREAD_SECTION_START -->

最前面的手动内容（可以写在这里）

<!-- WEREAD_HIGHLIGHT_1_START -->
<div class="weread-card" data-position="1">
  <span class="weread-text">第一条划线内容</span>
</div>
<!-- WEREAD_HIGHLIGHT_1_END -->

第一条后面的手动内容（可以写在这里）

<!-- WEREAD_HIGHLIGHT_2_START -->
<div class="weread-card" data-position="2">
  <span class="weread-text">第二条划线内容</span>
</div>
<!-- WEREAD_HIGHLIGHT_2_END -->

<!-- WEREAD_SECTION_END -->
```

### 手动内容穿插规则

在 `<!-- WEREAD_SECTION_START -->` 和 `<!-- WEREAD_SECTION_END -->` 之间：

- **最前面**的内容（在第一条划线之前）：保留
- **两条划线之间**的内容：保留，对应到前一条划线之后
- **不要**修改 `<!-- WEREAD_HIGHLIGHT_N_START/END -->` 标记
- **不要**修改 `<div class="weread-card">` 内部的划线内容

> 💡 简单说：划线之间可以自由写你的感想、评论，下次同步不会被覆盖。

## 自定义样式

编辑 `source/_data/weread.styl` 可以自定义划线卡片样式。

内置 4 种主题（在样式文件中修改 class 切换）：

- `.weread-card` - 默认主题（暖色渐变卡片）
- `.weread-card.theme-warm` - 温馨主题
- `.weread-card.theme-minimal` - 极简主题（左侧竖线）
- `.weread-card.theme-dark` - 深色主题

## 常见问题

### Q: 登录过期了怎么办？
A: 重新执行 `npm run weread-login` 扫码登录即可。

### Q: 提示"章节匹配失败"怎么办？
A: 
1. 检查博客文件夹中是否有对应的 `.md` 文件
2. 确认文件名（去掉数字前缀后）是否和微信读书章节名完全一致
3. 在 `weread_mapping.yml` 的 `chapters` 中手动添加映射

### Q: 可以只同步某几本书吗？
A: 可以。把不需要同步的书的 `sync` 设为 `false` 即可。

### Q: 手动添加的内容会被覆盖吗？
A: 不会。只要写在划线区域内、划线标记之外的内容，都会被保留。

### Q: GitHub Actions 可以自动同步吗？
A: 不建议。GitHub Actions 的服务器无法访问微信读书（国内服务），会导致超时失败。**推荐本地运行同步后推送代码。**

### Q: 划线顺序是按什么排列的？
A: 按在微信读书原文中出现的顺序排列。

## 工作原理

1. **读取配置**：加载 `weread_mapping.yml`，筛选 `sync: true` 的书籍
2. **启动浏览器**：Playwright 启动 Chromium，加载保存的登录状态
3. **抓取划线**：依次访问每本书的详情页，解析 DOM 提取划线内容和章节
4. **章节匹配**：根据文件名自动匹配，失败则查手动映射表
5. **更新文件**：读取 Markdown 文件，提取已有手动内容，重建划线区域，写回文件
6. **输出报告**：统计处理数量、错误信息，输出到控制台
