## 日常写作流程

### 第一步：新建文章

```powershell
# 进入项目目录
cd e:\白雨

# 新建文章
hexo new "文章标题"

# 新建到指定分类（推荐）
hexo new post "分类名/文章标题"
```

文章会生成在 `source/_posts/` 目录下。

### 第二步：编辑文章

用 Markdown 编辑器打开生成的 `.md` 文件，填写内容。

**文章头部示例：**
```yaml
---
title: 文章标题
date: 2026-06-27 12:00:00
tags: 标签名
categories:
- [分类名]
---

# 正文从这里开始

你的文章内容...
```

### 第三步：本地预览

```powershell
# 启动本地服务器
hexo server
# 或
npm run dev
```

打开浏览器访问：http://localhost:4000

### 第四步：提交并部署

```powershell
# 提交代码
git add .
git commit -m "新增文章：xxx"

# 推送（自动触发 GitHub Actions 部署）
git push
# 或用带重试的脚本
.\push.ps1
```

推送后约 1-2 分钟，网站自动更新。

| 命令 | 说明 |
|------|------|

## 文章格式说明

### Front-Matter（文章头部配置）

```yaml
---
title: 文章标题              # 必填，文章标题
date: 2026-06-27 12:00:00    # 必填，发布时间
tags: 标签名                 # 可选，标签
categories:                   # 可选，分类
- [分类名称]
---
```

### Markdown 语法

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体** *斜体* ~~删除线~~

- 无序列表项1
- 无序列表项2

1. 有序列表项1
2. 有序列表项2

> 引用文本

`行内代码`

```代码块
多行代码
```

[链接文字](链接地址)

![图片描述](图片地址)
```

### 插入图片

**方法一：放在文章同目录**
```
source/_posts/我的分类/
├── 我的文章.md
└── images/
    └── photo.jpg
```

文章中引用：
```markdown
![图片描述](./images/photo.jpg)
```

**方法二：放在 source/images 目录**
```markdown
![图片描述](/images/photo.jpg)
```

## 分类与标签

### 分类（Categories）

分类是层级结构，用于组织文章内容。

**设置分类：**
```yaml
categories:
- [老饭骨的家常菜]
```

**分类目录结构：**
```
source/_posts/
├── Book-01/         # 理想国
│   ├── 01第一卷.md
│   └── cover.jpg    # 分类封面图
├── Book-04/         # 法国大革命
│   ├── 01回响.md
│   └── cover.jpg
└── Book-12/         # 老饭骨的家常菜
    ├── 01姜汁黄瓜.md
    ├── 06凉拌羊肉.md
    └── cover.jpg
```

每个分类目录下放一张 `cover.jpg` 作为封面图。

### 标签（Tags）

标签是扁平结构，用于给文章打标记。

**设置标签：**
```yaml
tags: 凉拌肉菜
```

**多个标签：**
```yaml
tags:
- 凉菜
- 羊肉
- 家常菜
```

## 部署发布

### 自动部署（推荐）

推送源码到 `source` 分支，GitHub Actions 自动构建并部署到 `main` 分支。

```powershell
git add .
git commit -m "更新内容"
git push
```

查看部署状态：

```powershell

hexo clean

hexo deploy
```

```powershell
# 带重试的推送
.\push.ps1

# 完整本地部署
.\deploy.ps1
```

## 常见问题

### Q: 本地预览看不到新文章？

A: 检查以下几点：
1. 文章是否在 `source/_posts/` 目录下
2. 文章日期是否是未来时间（`future: true` 才会显示）
3. 重启本地服务器：`hexo server`

### Q: 推送失败，网络连接错误？

A: 尝试以下方案：
1. 使用带重试的脚本：`.\push.ps1 -MaxRetries 10`
2. 配置代理：`git config --global http.proxy http://127.0.0.1:7890`
3. 使用 GitHub Desktop 客户端

### Q: 网站更新后还是旧内容？

A: 强制刷新浏览器缓存：
- Windows: `Ctrl + F5`
- Mac: `Cmd + Shift + R`

或者检查 GitHub Actions 是否部署成功。

### Q: 图片不显示？

A: 检查图片路径：
- 文章内相对路径：`./images/photo.jpg`
- 根目录绝对路径：`/images/photo.jpg`

### Q: 如何修改网站标题？

A: 编辑 `_config.yml`：
```yaml
title: 白雨馆主
subtitle: '世界和我爱着你'
description: '孑然一身，爱一个人'
author: 白雨馆主
```

### Q: 构建时报错怎么办？

A: 先清理缓存再构建：
```powershell
hexo clean
hexo generate
```

如果还有错误，查看错误信息，通常是：
- Markdown 语法错误
- 配置文件格式错误（YAML 缩进问题）
- 缺少依赖（运行 `npm install`）
