# 微信读书划线同步到 Hexo 博客 —— 实施方案

## 一、项目概述

### 1.1 目标

将微信读书中的书籍划线自动同步到 Hexo 博客的对应文章中，以卡片形式展示，支持手动穿插笔记，定时自动同步。

### 1.2 核心需求

| 需求项 | 具体要求 |
|--------|---------|
| 数据来源 | 微信读书划线 |
| 数据中转 | Notion（通过 weread2notion-pro） |
| 匹配准确率 | 100%（配置驱动，匹配失败不写入） |
| 书籍选择 | 手动白名单，只同步确认过的书 |
| 展示风格 | Card 卡片式，CSS 可配置主题 |
| 手动内容 | 可穿插在任意两条划线之间，增量更新不影响 |
| 排序方式 | 按原文出现顺序 |
| 同步方式 | 定时自动同步（GitHub Actions） |

### 1.3 风险控制

- 不直接爬微信读书，通过成熟开源项目 weread2notion-pro 对接
- Notion 作为中间缓冲层，数据有备份
- 匹配失败跳过，绝不猜测写入

## 四、详细实施步骤

### 第一阶段：Notion + weread2notion-pro 配置

#### 步骤 1.1：创建 Notion 集成（Integration）

1. 打开 https://www.notion.so/my-integrations
2. 点击 "New integration"
3. 填写名称（如 "weread-sync"）
4. 选择工作区
5. 勾选所需权限（读取、编辑、插入）
6. 保存生成的 `Internal Integration Token`（记为 `NOTION_TOKEN`）

#### 步骤 1.2：创建 Notion 数据库

需要创建两个数据库：

**数据库 A：书籍库（Books）**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Name | Title | 书名 |
| BookId | Text | 微信读书书籍 ID |
| Author | Text | 作者 |
| Cover | File | 封面 |
| Status | Select | 阅读状态 |
| Progress | Number | 阅读进度 |
| Rating | Number | 评分 |
| Category | Select | 分类 |
| ReadTime | Number | 阅读时长 |
| LastReadDate | Date | 最后阅读时间 |

**数据库 B：划线库（Highlights）**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Name | Title | 划线内容摘要 |
| Book | Relation → Books | 关联书籍 |
| Chapter | Text | 章节名 |
| Content | Text | 划线原文 |
| Position | Number | 章节内位置 |
| CreateTime | Date | 划线时间 |
| BookmarkId | Text | 划线唯一 ID |

#### 步骤 1.3：共享数据库给集成

1. 打开每个数据库
2. 点击右上角 "..." → "Connections" → "Connect to"
3. 选择刚才创建的集成
4. 两个数据库都要添加

#### 步骤 1.4：获取数据库 ID

数据库 URL 格式：
```
https://www.notion.so/{workspace}/{database_id}?v=...
```
`database_id` 是 32 位字符串。

记录两个数据库 ID：
- `WEREAD_BOOKS_DB_ID`
- `WEREAD_HIGHLIGHTS_DB_ID`

#### 步骤 1.5：配置 weread2notion-pro

1. Fork 或克隆 `weread2notion-pro` 仓库到你的 GitHub
2. 在仓库 Settings → Secrets and variables → Actions 中添加：
   - `NOTION_TOKEN`
   - `WEREAD_COOKIE`（从微信读书网页版获取）
   - `WEREAD_BOOKS_DB_ID`
   - `WEREAD_HIGHLIGHTS_DB_ID`
3. 启用 GitHub Actions
4. 手动运行一次工作流，验证数据能同步到 Notion

#### 步骤 1.6：获取微信读书 Cookie

1. 用浏览器打开 https://weread.qq.com
2. 扫码登录
3. 按 F12 打开开发者工具
4. 切换到 Network（网络）标签
5. 刷新页面，点击任意一个请求
6. 在 Request Headers 中找到 `Cookie`
7. 复制完整的 Cookie 值（很长一串）

> **注意**：Cookie 有效期一般为 1-3 个月，过期后需要重新获取。

### 第二阶段：同步脚本开发

#### 步骤 2.1：创建脚本目录结构

```
scripts/weread-sync/
├── index.js          # 主入口
├── notion.js         # Notion 数据拉取
├── matcher.js        # 书籍/章节匹配
├── markdown.js       # Markdown 文件处理（核心算法）
├── config.js         # 配置加载
└── utils.js          # 工具函数
```

#### 步骤 2.2：创建配置文件

**文件 1：`source/_data/weread_mapping.yml`（白名单 + 映射表）**

```yaml
# 微信读书 → 博客 的书籍和章节映射
# 只有配置在这里的书才会同步（白名单机制）

books:
  # 示例：理想国
  - weread_book_id: "替换为微信读书书籍ID"
    weread_book_title: "理想国"
    blog_book_dir: "Book-01"
    sync: true      # 是否同步，设为 false 可临时暂停
    chapters: []    # 自动匹配失败的章节手动配置在这里

  # 示例：老饭骨的家常菜
  - weread_book_id: "替换为微信读书书籍ID"
    weread_book_title: "老饭骨的家常菜"
    blog_book_dir: "Book-12"
    sync: true
    chapters:
      - weread_chapter: "姜汁黄瓜"
        blog_post_file: "01姜汁黄瓜.md"
```

**文件 2：`source/_data/weread_config.yml`（同步参数）**

```yaml
# 展示配置
highlight:
  theme: "default"      # 主题: default / warm / minimal / dark
  show_number: false    # 是否显示序号

# 排序配置
sort:
  by: "position"        # position（原文顺序） / time（划线时间）

# 同步配置
sync:
  mode: "incremental"   # incremental（增量） / full（全量）
  fail_on_error: false  # 匹配失败时是否中止（false = 跳过并报错）
```

#### 步骤 2.3：Notion 数据拉取模块（notion.js）

**功能：**
- 使用 `@notionhq/client` SDK
- 从划线库查询所有划线
- 按书籍 ID 和章节名分组
- 每组内按位置排序

**输出数据结构：**
```javascript
{
  [weread_book_id]: {
    bookTitle: "书名",
    chapters: {
      "章节名": [
        { position: 1, content: "划线内容1", bookmarkId: "xxx" },
        { position: 2, content: "划线内容2", bookmarkId: "yyy" },
        ...
      ]
    }
  }
}
```

#### 步骤 2.4：匹配模块（matcher.js）

**书籍匹配：**
- 输入：微信读书书籍 ID
- 查找：`weread_mapping.yml` 中的 `weread_book_id`
- 输出：对应的 `blog_book_dir` 和章节映射
- 找不到 → 跳过，记录错误

**章节匹配（两步法）：**

第一步：规则自动匹配
- 读取博客文件夹内所有 `.md` 文件
- 去掉文件名开头的数字编号（正则：`/^\d+/`）
- 去掉 `.md` 后缀
- 与微信读书章节名精确比对（完全相等才算匹配）

第二步：手动映射匹配
- 自动匹配失败 → 查 `chapters` 配置
- 找到 → 使用配置的 `blog_post_file`
- 还是找不到 → 跳过，记录错误

**匹配原则：宁可漏掉，也不错放。**

#### 步骤 2.5：Markdown 处理模块（markdown.js）—— 核心算法

**功能：**
- 读取 markdown 文件
- 提取划线区域
- 解析已有划线和手动内容
- 重新构建划线区域
- 写回文件

**标记格式：**
```markdown
<!-- WEREAD_SECTION_START -->
  ...（划线和手动内容交替排列）...
<!-- WEREAD_SECTION_END -->
```

单条划线标记：
```markdown
<!-- WEREAD_HIGHLIGHT_N_START -->
<div class="weread-card" data-position="N">
  <span class="weread-text">划线内容</span>
</div>
<!-- WEREAD_HIGHLIGHT_N_END -->
```

**算法步骤：**

1. **提取区域**：用正则找到 `WEREAD_SECTION_START/END` 之间的内容
2. **解析片段**：将区域内容分割成交替的"划线片段"和"手动内容片段"
3. **建立手动内容映射**：
   ```javascript
   manualMap = {
     0: "所有划线最前面的内容",
     1: "第1条划线后面的内容",
     2: "第2条划线后面的内容",
     ...
   }
   ```
4. **重新构建区域**：
   - 遍历新划线列表（1, 2, 3, ... N）
   - 每条划先生成标记 + HTML
   - 如果 `manualMap[i]` 有内容，追加在该划线后面
   - 如果 `manualMap[0]` 有内容，放在最前面
5. **写回文件**：替换旧的划线区域

**安全性校验（同步前必须通过）：**
- SECTION 标记必须成对
- 每个 HIGHLIGHT_N_START 必须有对应的 HIGHLIGHT_N_END
- 序号必须连续（1, 2, 3, ...）
- 校验失败 → 不修改文件，输出错误

#### 步骤 2.6：主入口（index.js）

**执行流程：**
1. 加载配置（mapping + config）
2. 从 Notion 拉取划线数据
3. 遍历每本书：
   - 检查是否在白名单中（sync: true）
   - 不在 → 跳过
4. 遍历每章：
   - 匹配章节（自动 + 手动）
   - 匹配失败 → 记录错误，跳过
   - 匹配成功 → 处理 markdown 文件
5. 生成同步报告
6. 退出码：有错误但不致命 → 0；致命错误 → 1

**同步报告输出：**
```
=== 微信读书同步报告 ===
处理书籍: 3 本
处理章节: 15 章
新增划线: 8 条
修改文件: 5 个
匹配失败: 2 个
  - 理想国 / 柏拉图的生平（无对应章节文件）
  - 老饭骨的家常菜 / 凉拌木耳（无对应章节文件）
```

  "js-yaml": "^4.x.x"

### 第四阶段：GitHub Actions 自动化

#### 步骤 4.1：创建同步工作流

**文件：`.github/workflows/weread-sync.yml`**

```yaml
name: Weread Sync

on:
  schedule:
    - cron: '0 3 * * *'   # 每天凌晨 3 点运行（北京时间）
  workflow_dispatch:      # 支持手动触发

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          ref: source
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Run weread sync
        env:
          NOTION_TOKEN: ${{ secrets.NOTION_TOKEN }}
          WEREAD_HIGHLIGHTS_DB_ID: ${{ secrets.WEREAD_HIGHLIGHTS_DB_ID }}
          WEREAD_BOOKS_DB_ID: ${{ secrets.WEREAD_BOOKS_DB_ID }}
        run: npm run weread-sync

      - name: Check for changes
        id: git-check
        run: |
          git diff --exit-code || echo "changed=true" >> $GITHUB_OUTPUT

      - name: Commit and push
        if: steps.git-check.outputs.changed == 'true'
        run: |
          git config --local user.email "action@github.com"
          git config --local user.name "GitHub Action"
          git add -A
          git commit -m "docs: sync weread highlights"
          git push
```

#### 步骤 4.2：配置 GitHub Secrets

在仓库 Settings → Secrets 中添加：
- `NOTION_TOKEN`
- `WEREAD_BOOKS_DB_ID`
- `WEREAD_HIGHLIGHTS_DB_ID`

#### 步骤 4.3：与现有部署流程衔接

你现有的 deploy workflow 监听 source 分支的推送。新的同步 workflow push 到 source 分支后，会自动触发部署 workflow，完成 Hexo 构建和 GitHub Pages 发布。

## 五、日常使用指南

### 5.1 新增一本书到博客

1. 在微信读书中阅读并划线
2. 等待 weread2notion-pro 同步到 Notion
3. 在 Notion 中找到这本书，获取 `BookId`
4. 在 `source/_data/weread_mapping.yml` 中添加书籍配置
5. 在博客对应文件夹中创建章节 markdown 文件（如果还没有）
6. 等下次定时同步，或手动触发 GitHub Actions

### 5.2 添加手动笔记

1. 打开对应章节的 markdown 文件
2. 找到 `<!-- WEREAD_HIGHLIGHT_N_END -->` 标记
3. 在它后面写你想加的内容（支持 markdown 语法）
4. 保存、提交、部署

> **注意**：不要修改 `WEREAD_HIGHLIGHT_*_START` 和 `WEREAD_HIGHLIGHT_*_END` 标记本身。

### 5.3 更换展示主题

1. 编辑 `source/_data/weread_config.yml`
2. 修改 `highlight.theme` 的值（`default` / `warm` / `minimal` / `dark`）
3. 重新同步或手动触发

### 5.4 自定义样式

1. 编辑 `source/_data/weread.styl`
2. 修改对应主题的样式
3. 保存后重新部署

### 5.5 临时暂停某本书的同步

1. 编辑 `source/_data/weread_mapping.yml`
2. 将对应书籍的 `sync` 改为 `false`
3. 保存提交

### 5.6 更新微信读书 Cookie

当同步失败、提示登录过期时：
1. 重新登录微信读书网页版
2. 获取新的 Cookie
3. 更新 GitHub Secrets 中的 `WEREAD_COOKIE`
4. 重新运行 weread2notion-pro 的工作流

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|---------|
| 微信读书 Cookie 过期 | 高（1-3 个月） | 中（Notion 同步停止） | 监控告警 + 手动更新 Cookie |
| weread2notion-pro 接口变动 | 中 | 中（Notion 同步停止） | 关注项目更新 + 备用方案 |
| Notion API 限流 | 低 | 低（每天同步一次够用） | 加请求间隔 |
| 标记被误删导致解析失败 | 中 | 低（仅影响单篇） | 校验失败不修改 + 错误报告 |
| 章节匹配不上 | 中 | 低（跳过并报错） | 手动补充映射配置 |
